JiBX/WS: Starter

This project contains a simple example of running JiBX/WS.

Code Structure

For an overview of the folder structure, see Getting Started.

Building and Running

This application can be built and run using Maven, Gradle or Ant. The Maven and Gradle builds are the easiest to use, since they automatically start a Jetty server to run the web service.

Maven

Command Description
mvn jetty:run Builds the web service, starts Jetty on port 8080 and serves the project. Uses the JiBX Maven Plugin and Maven Jetty Plugin.
Alternatively, you could build a WAR file using mvn package if you wish to deploy to an alternative application server.
mvn exec:java Runs the client application, configured by default to connect to http://localhost:8080/welcome-service. Uses the Exec Maven Plugin.

Gradle

Requires Gradle v1.0-milestone-1 or later.

Command Description
gradle jettyRun Builds the web service, starts Jetty on port 8080 and serves the project. Uses the Gradle Jetty Plugin.
Alternatively, you could build a WAR file using gradle war if you wish to deploy to an alternative application server.
gradle run Runs the client application, configured by default to connect to http://localhost:8080/welcome-service. Uses the Gradle Application Plugin.

Ant

The Ant build creates a WAR file that you can deploy to your application server. If using Tomcat, it includes a target to deploy the WAR file to the Tomcat webapps folder.

Command Description
ant war Builds jibx-ws-starter.war
ant deploy-tomcat Copies jibx-ws-starter.war to CATALINA_HOME/webapps. You will need to have set the CATALINA_HOME environment variable to the location of your Tomcat installation prior to running this command. Depending on your Tomcat setup, you may need to perform additional steps (eg. restart Tomcat) to start the web application.
ant run Runs the client application, configured by default to connect to http://localhost:8080/jibx-ws-starter/welcome-service.

Expected Results

The web client sends a name (by default "World") to the web service. The service returns a greeting to say Hello to the client. The client will print out the server's response "Hello World!".
